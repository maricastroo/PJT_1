MUDANÇAS — ANTI-OVERFITTING E VOTING ENSEMBLE

CONTEXTO

Os modelos apresentavam gap grande entre acurácia de treino e validação/teste,
indicando overfitting. As mudanças abaixo atacam esse problema e adicionam um
novo arquivo de comparação de estratégias de ensemble.


ARQUIVO: models.py

ResNet-50 — correção de bug + cabeça própria

  Problema: a cabeça classificadora usava nn.Linear(512, 256), mas o backbone
  ResNet-50 produz vetores de 2048 dimensões. Isso causava erro em runtime.

  Solução: criada a classe ResNetHead(nn.Module) com forward() explícito:

    BatchNorm1d(2048) -> Linear(2048->256) -> ReLU -> Dropout(0.5) -> Linear(256->2)

  - BatchNorm1d normaliza as ativações antes das lineares, reduzindo dependência
    de padrões específicos do treino
  - Dropout aumentado de 0.2 para 0.5 para forçar mais redundância
  - in_features lido dinamicamente de model.fc.in_features (sem hardcode)


EfficientNet-B3

  - Dropout aumentado de 0.3 para 0.5 na cabeça classificadora


VGG-16

  - Adicionado BatchNorm1d(512) entre o primeiro bloco Linear(25088->512) -> ReLU
    e o Dropout, estabilizando o treinamento nas camadas densas


ARQUIVO: train_kfold.py

Label Smoothing

  Antes:  nn.CrossEntropyLoss(weight=class_w)
  Depois: nn.CrossEntropyLoss(weight=class_w, label_smoothing=0.1)

  Com label_smoothing=0.1 o modelo é penalizado por ficar 100% confiante nas
  predições de treino. Em vez de empurrar a probabilidade da classe correta para
  1.0, o alvo suavizado fica em 0.9. Isso reduz a confiança excessiva que é
  característica do overfitting.

ARQUIVO: voting.py

Após treinar os 3 modelos e gerar os JSONs em resultados/, execute:

    python voting.py

O script carrega os resultados de ResNet-50, EfficientNet-B3 e VGG-16 e
compara três estratégias de fusão usando P(maligno) de cada modelo:

  SOMA
    P_final = (p_resnet + p_effnet + p_vgg) / 3
    Média aritmética. Baseline robusto.

  PRODUTO
    P_final = (p1 * p2 * p3) / (p1*p2*p3 + (1-p1)*(1-p2)*(1-p3))
    Produto normalizado. Exige que os 3 modelos concordem para classificar
    como maligno.

  MAX
    P_final = max(p_resnet, p_effnet, p_vgg)
    Basta um modelo confiante para empurrar a decisão.

  Em todos os casos:
    P(maligno) >= 0.5  ->  Maligno
    P(maligno) <  0.5  ->  Benigno
    P(benigno) = 1 - P(maligno)

  Saída no terminal: tabela comparando AUC-ROC, F1-macro e acurácia de cada
  estratégia, com declaração do melhor método por AUC-ROC.

  Saída em arquivo: resultados/voting/results.json
